# 닐슨 코리아 시청기록 편집규칙 검증 연구

[![R](https://img.shields.io/badge/R-4.x-blue)](https://www.r-project.org)
[![Type](https://img.shields.io/badge/유형-산학협력-green)]()
[![Data](https://img.shields.io/badge/데이터-대외비-red)]()

> 산학협력 | 3인 팀 | 하루 평균 약 3,500가구 · 8일 · 약 5% 불일치율 · 5종 불일치 패턴 식별

---

## 🎯 핵심 발견

* 7일간 약 5%의 불일치율 확인 — 편집 규칙 적용 과정에서 유효 시청 기록 누락 및 잘못된 적격·부적격 판정 사례 발견
* 5종 불일치 패턴 분류 및 원인 분석 완료 — 채널 코드 불일치 패턴이 전체 불일치 중 가장 높은 비중 차지
* 패턴별 예시 케이스 추출 후 담당자와 공동 원인 분석 — 일부 패턴 발생 경로 특정 및 해결

---

## 📌 프로젝트 배경

* 시청률 데이터는 방송사 콘텐츠 기획·광고주 매체 집행 의사결정에 직접 영향을 미치는 핵심 지표 — TV 시청 기록은 패널 가구 미터기를 통해 원시 로그로 수집되며, 조사기관 내부 편집 규칙을 거쳐 분석 가능한 집계 데이터로 변환
* 편집 규칙 적용 과정은 외부에서 검증하기 어려운 블랙박스 구조 — 변환 과정에서 기록 누락이나 오매핑이 발생해도 외부에서는 발견하기 어려움
* 산학협력을 통해 원시 미터기 데이터와 편집 규칙 문서를 직접 제공받아, 연구팀이 독립적으로 알고리즘을 구현하고 공식 집계 데이터와 비교 검증

---

## 🔍 분석 설계

**분석 질문**

닐슨 코리아의 편집 규칙 알고리즘이 원시 미터기 로그를 공식 집계 데이터로 일관되게 변환하고 있는가. 변환 과정에서 유효한 시청 기록이 누락되거나 잘못 처리되는 사례가 존재하는가.

**접근 방식**

연구팀이 편집 규칙을 R로 독립 구현한 결과와 닐슨 측 공식 집계 데이터를 레코드 단위(가구 ID·채널 코드·시청 시작·종료 시각 기준)로 비교해 불일치 패턴 분류 및 원인 분석.

---

## ⚠️ 문제 상황

조사기관이 제공하는 최종 시청 데이터는 외부에서 생성 과정을 검증하기 어려운 구조. 편집 규칙 적용 기준이 내부 알고리즘에 의존해, 변환 과정의 오류를 외부에서는 발견하기 어려움.

| 문제 | 내용 |
|------|------|
| 검증 불가 구조 | 원시 로그에서 집계 데이터로의 변환 과정이 외부에 공개되지 않아 오류 발생 여부 확인 어려움 |
| 오류의 실질적 영향 | 시청 기록 오차는 시청률·점유율 산출에 직접 영향 — 방송사·광고주 의사결정 오류로 이어질 수 있음 |
| 편집 규칙 재현 복잡도 | 비정형 초 단위 로그 기반으로 다종 이상 패턴 탐지 규칙을 정확히 재현하는 난이도 |

---

## 🛠️ 분석 과정

### 1️⃣ 원시 로그 파싱 & 정형화

* 텍스트 형태의 원시 미터기 로그를 R로 파싱해 가구 ID·채널 코드·시청 시작·종료 시각 기준 데이터프레임으로 정형화 — 연구실 표준 분석 환경으로 동일 도메인 선행 연구에서 검증된 언어 채택, 팀 내 코드 공유 목적에도 적합

**결과:** 비정형 초 단위 로그 → 레코드 단위 정형 데이터프레임 변환 완료

---

### 2️⃣ 편집 규칙 3종 R 구현

* 무조작·특정 시간대 시청 등 비정상 시청 패턴을 탐지하는 이상 패턴 탐지 규칙 3종 구현 — 4종 편집 규칙 중 서버 동기화 시간 관련 규칙 1종은 검증에 필요한 정보 미제공으로 검증 범위 제외

**결과:** 편집 규칙 3종 독립 구현 완료 — 서버 동기화 관련 1종은 검증 범위 제외

---

### 3️⃣ 불일치 패턴 분류 & 원인 분석

* 연구팀 구현 결과와 공식 집계 데이터를 레코드 단위로 비교해 5종 불일치 패턴 분류 — 동일 기준으로 inner join 및 anti join 적용해 완전 일치·단방향 존재·시각/채널 불일치 유형 분리
* 불일치 패턴별 예시 케이스를 2~3건씩 추출해 닐슨 담당자와 공동 원인 분석 — 담당자 확인 가능 범위 내 원인 특정 및 일부 해결

**결과:** 5종 불일치 패턴 분류 완료 — 채널 코드 불일치가 전체 불일치 중 최다 비중, 일부 패턴 원인 특정 및 해결

## 📊 핵심 결과

### 5종 불일치 패턴

| 패턴 | 내용 |
|------|------|
| 공식 집계 데이터에만 존재 | 연구팀 구현 결과에는 없지만 공식 집계 데이터에 포함된 시청 기록 — 이상 패턴 탐지 규칙 미적용 및 미등록 채널 시청 기록 잔존 케이스 확인 |
| 연구팀 결과에만 존재 | 공식 집계 데이터에는 없지만 연구팀 구현 결과에 포함된 시청 기록 — 전처리 과정에서 유효 시청 기록이 누락된 케이스 확인 |
| 시작 시각 불일치 | 채널·종료 시각은 일치하지만 시청 시작 시각이 다르게 집계된 기록 — 다중 기기 환경에서 시청 기록 혼동 케이스 포함 |
| 종료 시각 불일치 | 채널·시작 시각은 일치하지만 시청 종료 시각이 다르게 집계된 기록 — 수분~수십 분 규모의 시청 시간 누락 케이스 확인 |
| 채널 코드 불일치 | 시청 시간은 일치하지만 채널 코드가 다르게 집계된 기록 — 전체 불일치 중 가장 높은 비중, 채널 식별 코드 이력 관리와 연관 |

## 💡 적용 가능성

* 방송 시청 조사기관의 편집 규칙 검증 체계 정례화
* 외부 감사·제3자 검증 파이프라인으로 시청 데이터 신뢰성 확보
* 원시 미터기 로그와 집계 데이터 간 자동 일관성 검사 도구로 확장 가능
* 동일 검증 프레임워크를 VOD·OTT 시청 데이터로 적용 가능

## 📈 한계점 및 향후 연구 방향

**한계점**

* 서버 동기화 시간 관련 편집 규칙 1종은 검증 범위 제외 — 해당 규칙의 적용 결과 비교 불가
* 제공된 정보 범위 내에서 검증 수행 — 일부 불일치 패턴의 원인은 내부 시스템 정보 없이 특정 불가

**개선 방향**

* 단기: 로그 구분 기준 정보 추가 제공 시 편집 규칙 적용 범위 확장 가능
* 중기: 채널 식별 코드 이력 정보 제공 시 채널 코드 불일치 패턴 원인 추적 정확도 향상 가능. 미등록 채널 분류 기준 명확화 시 해당 패턴 검증 범위 확장 가능
* 장기: 이번 연구에서 구축한 비교 검증 프레임워크를 정기 자동화 파이프라인으로 확장 가능

## 👥 역할

* 3인 팀에서 전처리 파이프라인 및 편집 규칙 구현 총괄 — 데이터 전처리, 알고리즘 구현, 비교 검증, 보고서 작성
* 비정형 초 단위 미터기 로그를 R로 파싱해 분석 가능한 데이터프레임으로 정형화
* 5종 불일치 패턴 분류 및 닐슨 담당자와의 원인 분석 회의 주도
