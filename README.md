## 📌 프로젝트 개요

Nielsen Korea 가구 시청 원시 미터기 데이터에 **4개 편집규칙 알고리즘을 R로 직접 구현**하고, 닐슨 공식 TX3 데이터와 **레코드 단위로 비교해 불일치 패턴 5종을 식별**한 데이터 품질 검증 연구입니다.

"블랙박스 알고리즘 재현 → 오류 발견 → 개선안 제시" 방식으로 기관 데이터 파이프라인의 신뢰성을 검증했습니다.

## 🔍 발견된 불일치 패턴 5종

| 패턴 | 설명 | 주요 원인 |
|------|------|---------|
| **TX3Only** | 닐슨 공식 TX3에는 있으나 구현 결과에 없음 | Playback 구분 로직 부재 |
| **Df_Only** | 구현 결과에는 있으나 TX3에 없음 | 처리 기준 차이 |
| **StartMismatched** | 시청 시작 시간 불일치 | 시각 보정 기준 불명확 |
| **EndMismatched** | 시청 종료 시간 불일치 | 비정상 종료 보정 누락 |
| **ChnMismatched** | 채널 코드 불일치 | 아리아나 코드 이력 관리 미비 |

## ⚠️ 주요 오류 사례

### 사례 1: 시청시간 최대 1시간 20분 누락
```
[원인] Playback(다시보기) 시청과 일반 시청 구분 로직 부재
       → Playback 시청 레코드가 편집규칙 적용에서 누락됨
[영향] 하나의 가구에서 단일 시청 세션 1시간 20분이 공식 TX3에서 미집계
[개선안] Playback 구분 플래그 추가 후 별도 처리 로직 적용
```

### 사례 2: 채널 코드 오매핑
```
[원인] 채널명 변경 이력(아리아나 → 다른 채널명) 미반영
       → 채널 코드 매핑 테이블이 구버전으로 유지됨
[영향] 해당 채널 시청 기록이 다른 채널 코드로 집계됨
[개선안] 채널 코드 변경 이력 DB 구축 및 정기 갱신 프로세스 수립
```

## 💡 개선안 제안 (3종)

1. **Playback 구분 로직 추가**: 일반 시청과 다시보기 시청 분리 처리
2. **아리아나 코드 이력 관리**: 채널 코드 변경 이력 DB 구축
3. **Chn 0 처리 기준 정비**: 미분류 채널(Chn=0)에 대한 명시적 처리 규칙 수립

## ✨ 분석 의의

KT 그룹 시청률 검증 연구와 함께 **"방송 시청률 데이터 품질 검증 시리즈"** 를 구성합니다.

두 연구의 공통 접근법:
> **블랙박스 알고리즘 재현 → 실제 데이터와 비교 → 오류 발견 → 개선안 제시**

단순 수치 분석이 아닌, 기관 데이터 파이프라인 자체의 신뢰성을 비판적으로 검토하는 역량을 체득한 프로젝트입니다.
